# 🏛️ تطبيق بوابة جامعة أنطاكيا السورية الخاصة (ASU)

تطبيق الويب الرسمي لـ **جامعة أنطاكيا السورية الخاصة (Antioch Syrian University - ASU)**. يُمثّل هذا المشروع المنصة الرقمية المتكاملة للجامعة، حيث يجمع بين واجهة تعريفية حديثة، ونظام إدارة أكاديمي تفاعلي مخصص للطلاب وأعضاء الهيئة التدريسية. تم بناء المنصة باستخدام أحدث تقنيات تطوير الويب لتقديم تجربة مستخدم سريعة، مستقرة، ومتجاوبة بالكامل تليق بالصرح الأكاديمي للجامعة في معرة صيدنايا.

---

## 🌟 الميزات الرئيسية (Key Features)

*   **🌐 دعم ثنائي اللغة والاتجاهات بالكامل (i18n & RTL/LTR):**
    *   نظام داخلي مخصص يعتمد على `React Context` للتبديل السلس بين اللغتين العربية (RTL) والإنجليزية (LTR).
    *   تحديث ديناميكي لخصائص مستند HTML والخطوط (استخدام خطوط `Tajawal` و `Cairo` للغة العربية و `Inter` للغة الإنجليزية).
*   **🎓 دليل الكليات والتخصصات ومستكشف البرامج الأكاديمية (Program Finder):**
    *   عرض متكامل لكليات الجامعة (الهندسة، الصيدلة، طب الأسنان، العلوم الإدارية والاقتصادية، الآداب والعلوم الإنسانية، الحقوق).
    *   مستكشف برامج أكاديمي ذكي يسمح بتصفية التخصصات والبحث عنها حسب المعدلات المطلوبة، ونوع الشهادة الثانوية، والكلية.
*   **📊 لوحات تحكم تفاعلية متكاملة (Student & Instructor Dashboard):**
    *   لوحات تحكم ذكية وشاملة للطلاب لمتابعة الدرجات والجدول الدراسي والرسوم والمعدل التراكمي (GPA).
    *   رسوم بيانية تفاعلية باستخدام مكتبة `Recharts` لمراقبة الأداء الدراسي والساعات المنجزة.
*   **✨ تجربة مستخدم ممتازة ورسوم متحركة (Premium UI/UX & Animations):**
    *   تأثيرات حركة متميزة وانسيابية عند التمرير بالاعتماد على `Framer Motion` وتمرير فيزيائي فائق السلاسة بـ `Lenis Scroll`.
    *   تصميم واجهات احترافي وعصري يعتمد على نظام ألوان HSL و OKLCH المطور في Tailwind CSS v4.
*   **🏛️ الحياة الجامعية والبحث العلمي (Campus Life & Research):**
    *   مساحات خاصة لعرض النشاطات الجامعية، الأخبار، الإعلانات الهامة للطلاب، والبحوث العلمية الصادرة عن الهيئة التدريسية.

## 🛠️ التقنيات المستخدمة (Tech Stack)

### الإطار الأساسي (Core)
*   **Next.js 15+ (App Router)** - إطار العمل الأساسي لتطوير الويب وبناء الصفحات وتهيئتها لمحركات البحث (SEO).
*   **React 19** - مكتبة بناء الواجهات البرمجية وتفاعليتها.
*   **TypeScript** - لضمان سلامة كتابة الكود وتتبع الأخطاء أثناء التطوير بشكل صارم.

### التصميم والواجهات (Styling & UI)
*   **Tailwind CSS v4** - للتنسيق السريع وتخصيص السمات (الألوان المتطورة OKLCH).
*   **Shadcn/UI (Radix UI)** - لتوفير العناصر الرسومية الجاهزة والوصولية الميسرة (Accessibility).
*   **Lucide React** - حزمة الأيقونات الموحدة للمشروع.

### الرسوم التفاعلية والتحكم (Animations & Charts)
*   **Framer Motion 12** - لإضافة الحركات التفاعلية وتأثيرات الدخول المعقدة.
*   **Lenis Scroll** - لتوفير التمرير الانسيابي الفاخر (Smooth Inertia Scrolling).
*   **Recharts** - لعرض الإحصائيات الأكاديمية والبيانات للطلاب بصرياً.

### النماذج والتحقق (Forms & Validation)
*   **React Hook Form** - لإدارة الحقول وحالة النماذج بكفاءة عالية.
*   **Zod** - للتحقق من صحة المدخلات وتكامل البيانات.

## 🚀 تشغيل وتهيئة المشروع (Setup & Installation)

لتشغيل المشروع محلياً على جهاز التطوير الخاص بك، يرجى تتبع الخطوات التالية:

### المتطلبات المسبقة
تأكد من تنصيب **Node.js** (الإصدار 18 فما فوق يوصى به) و مدير الحزم **npm** أو **pnpm**.

### 1. تثبيت الاعتمادات (Install Dependencies)
قم بفتح الطرفية (Terminal) في المجلد الرئيسي للمشروع، وشغّل الأمر التالي لتثبيت الحزم المطلوبة:
```bash
npm install
# أو إذا كنت تستخدم pnpm
pnpm install
```

### 2. تشغيل خادم التطوير (Run Development Server)
بعد اكتمال التثبيت، شغّل الأمر التالي لبدء خادم التطوير المحلي:
```bash
npm run dev
# أو
pnpm dev
```
سيكون التطبيق متاحاً على الرابط المباشر: `http://localhost:3000`

### 3. بناء النسخة الإنتاجية (Build for Production)
لبناء حزمة التطبيق المحسنة وتهيئتها للرفع للإنتاج:
```bash
npm run build
```

## 📜 معايير البرمجة والتطوير (Coding Guidelines & Standards)

يلتزم مجتمع المطورين للمشروع بالمعايير التالية لضمان نظافة الكود واستقراره (انظر التفاصيل الكاملة في [AI_RULES.md](file:///mnt/3dac2f44-6960-4b29-b035-9c2b4a87fb9d/PC_Work/ASU/ASU_Website/ASU-main/AI_RULES.md)):

*   **قاعدة المكون الأحادي:** يجب إنشاء ملف منفصل لكل مكون برمجي (Component). لا تقم بتعريف أكثر من مكون في نفس الملف إطلاقاً.
*   **حجم الملفات:** يجب الحفاظ على حجم المكونات البرمجية قصيرة وبسيطة (أقل من 100 سطر برمجي). إذا زاد الحجم، قم بتقسيم المكون إلى مكونات أصغر قابلة لإعادة الاستخدام.
*   **إدارة النصوص والترجمة (i18n):**
    *   يُمنع كتابة النصوص النصيّة (Hardcoded Strings) داخل المكونات مباشرة.
    *   استخدم دائماً خطاف `useLanguage` ودالة `t()` لعرض النصوص بالاعتماد على مفتاح الترجمة الموحد.
*   **التوافقية والاتجاهات:** استخدم خاصية `dir` من سياق اللغة وتأكد من اختبار الواجهات البرمجية وتجاوبها بشكل كامل في حالتي `ar` (من اليمين لليسار) و `en` (من اليسار لليمين).
*   **مكونات واجهة المستخدم:** يجب الاعتماد على المكونات الموجودة في `components/ui` وفي حال لم يكن المكون موجوداً قم بإضافته من Shadcn/UI مع مراعاة الوصولية والتحكم الكامل بلوحة المفاتيح.
